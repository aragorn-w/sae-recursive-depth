# sae-recursive-depth

**How Deep Does the Rabbit Hole Go? Testing the Limits of Recursive Feature Decomposition via Multi-Depth Meta-Sparse Autoencoders on Gemma-2-2B and GPT-2 Small**

Aragorn Wang, Colorado School of Mines.

Repo: https://github.com/aragorn-w/sae-recursive-depth

## Final paper

The compiled writeup is at [`docs/WRITEUP.pdf`](docs/WRITEUP.pdf). The Markdown source is [`docs/WRITEUP.md`](docs/WRITEUP.md) and the pandoc-emitted LaTeX is [`docs/WRITEUP.tex`](docs/WRITEUP.tex). To rebuild the PDF from source:

```bash
bash docs/scripts/render.sh pdf
```

The render script invokes pandoc → xelatex → biber → xelatex → xelatex against the IEEEtran conference template at `docs/scripts/ieeetran.template.tex`. Figures are regenerated from `data/results.tsv` via `bash docs/scripts/render.sh figures`.

## Status

80 of 80 non-stretch matrix rows in `EXPERIMENTS.yaml` completed (`latest=ok` in `data/results.tsv`). One stretch row (`simplestories_stretch`) failed and was deferred per the project spec. Numeric headlines and figures are in `docs/WRITEUP.pdf`.

## Contributions

- A locked, committed execution matrix (`EXPERIMENTS.yaml`) plus an append-only ground-truth log (`data/results.tsv`) that decouples claims in the paper from re-execution drift; every quantitative claim cites a row.
- An empirical depth-by-depth ladder for recursive meta-SAEs (depth 0–3, two base models, BatchTopK + JumpReLU + flat-width null) with cross-seed PW-MCC, MMCS, and autointerp scoring at each cell.

## Setup

Dependencies are managed with [uv](https://docs.astral.sh/uv/). Install uv if it isn't already present (`curl -LsSf https://astral.sh/uv/install.sh | sh`), then resolve and install the locked environment:

```bash
uv sync
```

This creates a project-local `.venv/` from `pyproject.toml` / `uv.lock`. Training and analysis additionally require:

- W&B login: `uv run wandb login`
- HuggingFace login for gated Gemma-2-2B access: `uv run huggingface-cli login`, then accept the Gemma license at https://huggingface.co/google/gemma-2-2b
- An NVIDIA GPU with CUDA available to PyTorch

Ad-hoc scripts run inside the managed environment with `uv run`, e.g. `uv run python scripts/evaluate_gates.py`.

## Reproducibility

The append-only `data/results.tsv` is the single source of truth: if a number in the writeup does not trace to a row and column there, it should not be in the writeup. Per-experiment artifacts and figures (checkpoints, per-figure source `.tsv` / `.csv` files, autointerp scores) are written under `data/artifacts/<experiment_id>/` during runs. Each figure's companion source-numbers file carries the generation timestamp, git commit SHA, analysis script path, and the set of `experiment_id` values consumed.

## Compiling the writeup to LaTeX

The full pandoc → xelatex → biber pipeline is wrapped in `docs/scripts/render.sh`:

```bash
bash docs/scripts/render.sh pdf       # rebuild docs/WRITEUP.pdf
bash docs/scripts/render.sh figures   # regenerate figures from data/results.tsv first, then rebuild
bash docs/scripts/render.sh tex       # emit docs/WRITEUP.tex only (no PDF)
bash docs/scripts/render.sh clean     # remove generated artifacts
```

`docs/WRITEUP.md` is the editable source. `docs/scripts/ieeetran.template.tex` is the pandoc template wrapping the official IEEEtran conference class; `docs/scripts/twocolumn_floats.py` post-processes pandoc's longtable / figure output into IEEE-compliant `table*` / `figure*` two-column floats.
